DESCRIPTION

t.rast.import.netcdf imports content of one or more NetCDF files into a GRASS GIS Space Time Raster Dataset (STRDS). NetCDF files are expected to follow the CF-convention. Files not adhearing to those standards are likely to fail to import.

Input URL(s) to NetCDF files can be provided in the input option as either a file, with one URL to a dataset per line, a comma-separated list of URLs or a single URL. "-" causes input to be taken from stdin.

The module works for both local and remote data (e.g. on a Thredds Server). Data can be imported (with r.in.gdal or r.import) or linked with r.external.

t.rast.import.netcdf can use GDALs Virtual Raster format (VRT) if data with Coordinate Reference systems that differ from the one of the current location are supposed to be linked only.

Starting with GRASS GIS version 8.0, different variables or subdatasets in a NetCDF file can be imported as "band reference" into one STRDS. To achieve this, a configuration file has to be provided in the bandref option, that maps subdatasets in the NetCDF file to GRASS GIS band references. Each line in that file must have the following format: input_netcdf_subdataset=grass_gis_band_reference The equal sign = is required. If a band references configuration file is provided, the import of subdatasets is limited to those subdatasets listed in that file.

KNOWN ISSUES

The VRT format is also used for NetCDF data that contains subdatasets, as import of those is currently not supported in r.external.

REQUIREMENTS

Support of band references is only available with GRASS GIS 8.0 or later. t.rast.import.netcdf uses the following non-standard Python modules:
  • numpy
  • gdal
  • cf-units

EXAMPLES

Link Sentinel-2 scenes from the Norwegian Ground Segment

# Choose Scenes to import (see also m.crawl.thredds module)
echo "https://nbstds.met.no/thredds/fileServer/NBS/S2A/2021/02/28/S2A_MSIL1C_20210228T103021_N0202_R108_T35WPU_20210228T201033_DTERRENGDATA.nc
https://nbstds.met.no/thredds/fileServer/NBS/S2A/2021/02/28/S2A_MSIL1C_20210228T103021_N0202_R108_T32VNL_20210228T201033_DTERRENGDATA.nc" > nc.txt

# Create a band reference configuraton file
echo "B1=S2_1
B2=S2_2" > bandref.conf

# Import data (link NetCDF files without downloading them)
t.rast.import.netcdf -l input=nc.txt output=S2A title=S2A basename=S2A \
description=S2A bandref=bandref.conf memory=2048 nprocs=2

Import Norwegian Climate data

# Choose dataset to import (see also m.crawl.thredds module)
echo "https://nbstds.met.no/thredds/fileServer/NBS/S2A/2021/02/28/S2A_MSIL1C_20210228T103021_N0202_R108_T35WPU_20210228T201033_DTERRENGDATA.nc
https://nbstds.met.no/thredds/fileServer/NBS/S2A/2021/02/28/S2A_MSIL1C_20210228T103021_N0202_R108_T32VNL_20210228T201033_DTERRENGDATA.nc" > nc.txt

# Create a band reference configuraton file
echo "tg=temperature_avg
tn=temperature_min" > bandref.conf

# Import data
t.rast.import.netcdf output=SeNorge title=SeNorge basename=SeNorge description=SeNorge \
input=https://thredds.met.no/thredds/fileServer/senorge/seNorge_1957/Archive/seNorge2018_2021.nc \
bandref=bandref.conf memory=2048 nprocs=2

Append to STRDS from previous imports

# Choose dataset to import (see also m.crawl.thredds module)

# Create a band reference configuraton file
echo "tg=temperature_avg
tn=temperature_min" > bandref.conf

# Import data
t.rast.import.netcdf output=SeNorge title=SeNorge basename=SeNorge description=SeNorge \
input=https://thredds.met.no/thredds/fileServer/senorge/seNorge_2020/Archive/seNorge2018_2021.nc \
bandref=bandref.conf memory=2048 nprocs=2 -a
